SQL Data Analysis Project

Project Overview

This project demonstrates the use of SQL queries to extract meaningful business insights from an e-commerce dataset. The analysis focuses on retrieving, filtering, grouping, and aggregating data to support data-driven decision-making.

Objective

The objective of this project is to apply fundamental SQL concepts to analyze transactional data and generate actionable insights.

Tools Used

* SQLite
* DB Browser for SQLite
* SQL
* CSV Dataset

Dataset Description

The dataset contains e-commerce order information, including:

* Order ID
* Order Date
* Customer ID
* Product
* Quantity
* Unit Price
* Payment Method
* Order Status
* Referral Source
* Total Price

SQL Concepts Applied

Data Retrieval

* SELECT

Data Filtering

* WHERE

Data Sorting

* ORDER BY

Data Grouping

* GROUP BY

Aggregation Functions

* COUNT()
* SUM()
* AVG()

Queries Performed

1. Total Number of Orders

Calculated the total number of orders in the dataset using COUNT().

2. Average Order Value

Determined the average value of customer orders using AVG().

3. Total Revenue

Calculated the overall revenue generated using SUM().

4. High-Value Orders

Filtered orders with a total price greater than a specified threshold using WHERE.

5. Most Ordered Products

Grouped products and counted the number of orders for each product using GROUP BY and COUNT().

6. Revenue by Product

Calculated revenue generated by each product using SUM() and GROUP BY.

7. Orders by Payment Method

Analyzed customer payment preferences by counting orders for each payment method.

8. Average Order Value by Referral Source

Compared average spending behavior across different referral sources.

Key Insights

* The dataset contains 1200 customer orders.
* Product performance varies significantly across categories.
* Revenue contribution differs among products.
* Payment methods show different levels of customer adoption.
* Referral sources influence customer purchasing behavior.
* Aggregation functions help summarize large volumes of transactional data efficiently.
